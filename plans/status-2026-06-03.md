# Status: agent-experiment

> **Period**: 2026-05-23 through 2026-06-03
> **Last updated**: 2026-06-03T19:18:30Z

## Summary
Released version 0.4.0 with the completed multi-stage framework split and began work on abstain-aware consensus in judge aggregation. All three major stages (Framework Split, ReEvaluator, JudgeExperiment) are complete with comprehensive test coverage. Stage 4 (abstain-aware consensus) is in progress but targets the agent-judge repository.

## What Was Accomplished

### 1. Version 0.4.0 Release
- **Released 0.4.0** (commit e12be9f) with all dependencies updated to released versions
- **Completed framework split**: Shared experiment infrastructure now decoupled from agent-specific execution via `ExecutionDetail` marker interface
- **Added ReEvaluator**: Post-hoc re-scoring of stored results without system re-invocation
- **Added JudgeExperiment**: Sibling typed experiment API for judge calibration alongside agent experiments

### 2. Development Version Management
- **Bumped to 0.5.0-SNAPSHOT** (commit 65b1846) for next development cycle
- **Updated dependencies**: Migrated all dependencies from `org.springaicommunity` to `io.github.markpollack` groupId

### 3. Infrastructure Improvements
- **Fixed RunLogManager logback dependency** (commit 4e9fd11): Moved logback references to lazy-loaded inner classes, making logback dependency optional rather than mandatory
- **Relaxed git dirty check** (commit 10e0be6): Changed from throwing errors to warnings for non-critical paths

### 4. Judge Aggregation Enhancement
- **Started abstain-aware consensus work** (commit 1b750b2): Addresses ABSTAIN judgment handling in `ConsensusStrategy` to align with `MajorityVotingStrategy` patterns
